# Task 011: 반응형 디자인 완성

## 개요

모바일 및 태블릿 뷰 최적화를 수행하고 다양한 화면 크기에서 최적의 사용자 경험을 제공합니다.

## 관련 파일

- `src/components/dashboard-content.tsx` - 대시보드 메인 컨텐츠
- `src/components/dividend-table.tsx` - 피벗 테이블
- `src/components/summary-cards.tsx` - 요약 카드
- `src/components/monthly-chart.tsx` - 월별 차트

## 수락 기준

- [ ] 모바일 뷰 (< 640px)에서 모든 컴포넌트가 정상 표시됨
- [ ] 태블릿 뷰 (640px - 1024px)에서 레이아웃이 최적화됨
- [ ] 피벗 테이블이 가로 스크롤로 처리됨
- [ ] 차트 크기가 뷰포트에 맞게 조정됨
- [ ] 터치 인터랙션이 원활함

## 구현 단계

### 1단계: 요약 카드 반응형 처리

- [ ] 모바일: 2x2 그리드
- [ ] 태블릿: 2x2 그리드
- [ ] 데스크톱: 4x1 그리드 (현재 상태)
- [ ] 카드 내용 폰트 크기 조정

### 2단계: 차트 반응형 처리

- [ ] ResponsiveContainer 설정 확인
- [ ] 모바일에서 Y축 레이블 간소화
- [ ] 툴팁 위치 조정
- [ ] 차트 높이 뷰포트 비율 조정

### 3단계: 피벗 테이블 반응형 처리

- [ ] 가로 스크롤 컨테이너 적용
- [ ] 고정 열 (종목명) 구현 고려
- [ ] 모바일에서 폰트 크기 축소
- [ ] 스와이프 힌트 UI 추가

### 4단계: 헤더 반응형 처리

- [ ] 모바일에서 세로 배치 (제목 + 연도 선택기)
- [ ] 제목 폰트 크기 조정
- [ ] 연도 선택기 풀 너비 (모바일)

### 5단계: 터치 인터랙션 개선

- [ ] 버튼/선택기 터치 영역 확대 (44px 이상)
- [ ] 스와이프 제스처 지원 (테이블)
- [ ] 탭 포커스 순서 확인

### 6단계: 접근성 개선

- [ ] ARIA 레이블 추가
- [ ] 키보드 네비게이션 확인
- [ ] 색상 대비 확인
- [ ] 스크린 리더 테스트

## 브레이크포인트 정의

```css
/* Tailwind 기본 브레이크포인트 */
sm: 640px   /* 모바일 -> 태블릿 */
md: 768px   /* 태블릿 중간 */
lg: 1024px  /* 태블릿 -> 데스크톱 */
xl: 1280px  /* 대형 데스크톱 */
```
